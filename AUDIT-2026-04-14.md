## Findings

### ✅ [HIGH] vecdex: `remove()` can produce one-directional edges (INV-VD2)

**WHERE**: `strata/src/vecdex/mod.rs` — `remove()` Phase 2 reconnection loop
**WHAT**: After adding a bidirectional edge (n↔candidate), `prune_neighbors(candidate)` may evict `n` from candidate's neighbor list, but `n`'s list already contains `candidate` and is not rolled back.
**WHY**: INV-VD2 — every edge (A,B) at layer L must exist in both neighbor lists.
**FIX**: After `prune_neighbors(candidate)`, verify the reverse edge survived; if not, remove `candidate` from `n`'s list symmetrically.

### ✅ [MEDIUM] versioning: `gc_dirty` not set before branch/commit mutations — crash orphans commits permanently

**WHERE**: `strata/src/versioned/map.rs` — `commit()`, `merge()`, `rollback_to()`
**WHAT**: `gc_dirty` is only set inside `decrement_ref`. Every mutating function creates commits or updates branch heads before `decrement_ref` is called (and `commit()` never calls it). Crash in any of these windows leaves an orphaned commit with `ref_count=1`, permanently unreclaimable by `gc()`.
**WHY**: Violates INV-V5 (dirty flag lifecycle) and INV-V1 (ref-count balance).
**FIX**: Set `gc_dirty = true` at the top of each mutating function; clear to `false` just before `Ok(...)`.

---

### ✅ [MEDIUM] btree: `shrink_root` discards COW intermediate node, permanently inflating surviving child's `ref_count`

**WHERE**: `strata/src/basic/persistent_btree/mod.rs` — `shrink_root` + `fix_underflow`
**WHAT**: Height reduction allocates `I'` wrapping merged child `M` (incrementing `M.ref_count`), then `shrink_root` drops `I'` without cleanup. `M.ref_count` stays permanently +1 higher.
**WHY**: Violates INV-BT5 (GC reachability). Incremental GC on `M` is broken until full `gc()`/`rebuild_ref_counts`.
**FIX**: In `shrink_root`, register the dropped node for lazy deletion and undo the spurious ref_count increment.

### ✅ [MEDIUM] btree: `release_node` uses `saturating_sub` — silently cascade-deletes on double-release

**WHERE**: `strata/src/basic/persistent_btree/mod.rs` — `release_node`
**WHAT**: `saturating_sub(1)` on `ref_count=0` yields 0, triggering full subtree cascade deletion without any diagnostic.
**WHY**: Makes under-release indistinguishable from legitimate final release; compounds with the shrink_root finding.
**FIX**: Add `debug_assert!(nr.ref_count > 0)` and guard the decrement.

### ✅ [MEDIUM] slotdex: spurious tier destruction on failed `remove`

**WHERE**: `strata/src/slotdex/mod.rs:232–247`
**WHAT**: The tier-shrinking loop runs unconditionally on every `remove()` call, before `exist` is checked. A no-op remove (key absent) still destroys degenerate tiers.
**WHY**: No-op operations must not mutate structural state.
**FIX**: Guard the shrinking loop and count updates with `if exist { ... }`.

### ✅ [MEDIUM] encoding: blanket `KeyEnDe` permits non-deterministic `HashMap`/`HashSet` keys

**WHERE**: `strata/src/common/ende.rs` — blanket `impl<T: Serialize> KeyEn for T`
**WHAT**: Covers `HashMap`/`HashSet` whose serialization order depends on random `SipHash` seed. A `Mapx<HashMap<…>, V>` compiles but silently breaks after process restart.
**WHY**: Pattern 6.1 — key lookup is byte-exact match.
**FIX**: Add a prominent `# Warning` doc block on `KeyEn`/`KeyEnDe` prohibiting non-deterministic types as keys.

### ✅ [MEDIUM] dagmap/raw: `insert()` silently converts empty value to deletion tombstone

**WHERE**: `strata/src/dagmap/raw/mod.rs` — `insert()`, `remove()`, `get()`
**WHAT**: `remove()` stores an empty byte slice as tombstone; `get()` treats empty values as `None`. `insert(k, b"")` is silently equivalent to `remove(k)`.
**WHY**: Hidden precondition on public API.
**FIX**: Add `debug_assert!(!value.as_ref().is_empty())` and document the prohibition.
