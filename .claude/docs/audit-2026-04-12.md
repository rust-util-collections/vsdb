## Findings

### [HIGH] slotdex: insert after deserialization corrupts tier caches and store

**Where**: `strata/src/slotdex/mod.rs:192-204`

**What**: The `insert` method directly reads from and writes to the tier cache
(`RefCell<BTreeMap>`) without calling `ensure_cache()` first. After
deserialization (via `from_meta` or postcard), the tier cache is empty
(`#[serde(skip)]`) while the tier store has correct data on disk. The first
insert reads 0 from the empty cache for the slot's floor-aligned bucket, writes
1 to both cache and store, overwriting the correct pre-existing count (which
could be any value). All subsequent queries will compute wrong skip counts from
the corrupted tier, returning incorrect page results.

The same issue affects `ensure_tier_capacity` (~line 569-574), which reads from
the top tier's cache to build a new tier — an empty cache produces an empty new
tier, losing all aggregated counts.

The `remove` method correctly calls `t.ensure_cache()` before reading the cache,
but `insert` does not.

**Why**: Encoding/serialization boundary bug (technical-patterns.md Category 6).
The `#[serde(skip)]` cache is not populated before use on the insert path,
violating tier count consistency.

**Fix**: Add `t.ensure_cache()` at the start of the tier update loop in `insert`
(before line 193), and at the start of the `ensure_tier_capacity` branch that
reads from `top.cache`. Add a regression test that deserializes a SlotDex then
inserts + queries.

### [MEDIUM] dagmap: prune_mainline leaks intermediate nodes' side-branch children

**Where**: `strata/src/dagmap/raw/mod.rs:258-309`

**What**: When `prune_mainline` merges a chain into genesis, only `self`'s
children are re-parented to genesis (lines 291-299) and only `self` is cleaned
up (lines 302-304). Intermediate nodes in `linebuf` (the "others" slice) are
never cleaned up: their data, their children, and their children's entire
subtrees persist on disk but become unreachable from any live node.

Concrete scenario: create `G -> P -> H` where `P` has a side-branch child `S`.
Call `H.prune()`. Result: `S` and its descendants remain on disk permanently,
unreachable from genesis.

**Why**: Violates orphan prevention — dead nodes accumulate unboundedly
(analogous to technical-patterns.md Pattern 1.3 structural sharing leak, adapted
to DAG context).

**Fix**: Before merging data, iterate each node in "others" and either destroy
their children or re-parent them to genesis. Then clear each intermediate node's
data, parent, and children fields.

### [LOW] dagmap: get_mut does not traverse parent chain, asymmetric with get

**Where**: `strata/src/dagmap/raw/mod.rs:226-232`

**What**: `get()` traverses up to 1024 parent links to find a key, but
`get_mut()` only checks `self.data`. A key visible via `get()` may return `None`
from `get_mut()` on the same node. The doc comment does not mention this
limitation.

**Why**: Likely intentional (COW semantics — mutations are local), but
undocumented asymmetry could mislead callers.

**Fix**: Add a doc comment clarifying that `get_mut` only operates on the node's
own data and does not traverse parents.

---

### [LOW] engine: unsafe blocks missing SAFETY comments

**Where**: `core/src/common/mod.rs:68`, `core/src/common/mod.rs:235`

**What**: Two `unsafe { env::set_var(...) }` calls lack `// SAFETY:` comments.
In Rust 2024 edition, `env::set_var` is unsafe because it is not thread-safe.
The calls are guarded by `LazyLock`/`HAS_INITED` ensuring single-execution, but
the safety reasoning is not documented.

**Why**: Project convention requires all unsafe blocks to have SAFETY comments.

**Fix**: Add SAFETY comments documenting the early-initialization assumption.

## Detailed Coverage per Subsystem

### Engine & Raw KV

**Files**: `core/src/common/engine/mmdb.rs`, `core/src/common/engine/mod.rs`,
`core/src/common/mod.rs`, `core/src/basic/mapx_raw/mod.rs`, `core/src/lib.rs`

- **Prefix isolation (INV-E1)**: Allocator is monotonic. Initial on-disk value
  = `RESERVED_ID_CNT + u64::MIN` = 40960000. `GLOBAL_COUNTER` uses
  `fetch_add(PREFIX_ALLOC_BATCH, AcqRel)` ensuring disjoint ranges per thread.
  On-disk ceiling persisted before any prefix from a batch is used. No recycle.
- **Shard routing (INV-E2)**: All code paths use `u64::from_le_bytes(prefix) %
  NUM_SHARDS` via the same `shard()` method. Verified in get, insert, remove,
  lazy_delete, iter, range, batch_begin.
- **Singleton init (INV-E4)**: `LazyLock<VsDB>` — exactly-once, blocking.
- **Cross-shard atomicity (INV-E5)**: `MmdbBatch` scoped to single
  `meta_prefix` (single shard). No cross-shard batches.
- **Iterator prefix bound (INV-E6)**: `iter()` scoped by `iter_with_prefix()`.
  `range()` adds `starts_with(&meta_prefix)` filter. `prefix_successor()` handles
  all-0xFF edge case by returning `None`.
- **Key construction (INV-E3)**: `make_full_key()` prepends 8-byte prefix
  consistently in all paths.

### Typed Collections

**Files**: `strata/src/basic/mapx/`, `strata/src/basic/mapx_ord/`,
`strata/src/basic/mapx_ord_rawkey/`, `strata/src/basic/orphan/`

- **Encoding determinism**: postcard for Mapx, big-endian order-preserving for
  MapxOrd, raw bytes for MapxOrdRawKey, empty-key singleton for Orphan. No
  HashMap/HashSet key types.
- **Type safety**: Typed wrappers encode before passing to raw layer, decode on
  retrieval. PhantomData markers maintain type-level safety.
- **Iterators**: Forward + reverse via `DoubleEndedIterator`. Mutable iterators
  use write-back-on-drop (DerefMut sets dirty flag, Drop writes to DB).
- **Entry API**: Raw pointer cast in mutually exclusive match arms — SAFETY
  comment valid, no aliasing.

### B+ Tree

**Files**: `strata/src/basic/persistent_btree/` (all .rs files)

- **COW integrity**: All 8 mutation paths (insert_leaf, insert_internal,
  remove_leaf, remove_internal, fix_underflow, borrow_left, borrow_right,
  merge_children) read-by-value via `self.node()` (decodes from storage),
  modify in-memory, allocate via `self.alloc()` (fresh NodeId). No in-place
  mutation of existing NodeIds.
- **Split boundaries**: Leaf split: mid = 16, left gets 16 keys, right gets 17,
  separator = first key of right. Internal split: median promoted, removed from
  both children.
- **Ref-counts**: `alloc()` increments child ref_counts. `release_node()`
  cascades decrements. `rebuild_ref_counts()` does full BFS reachability sweep.
- **Merge/rebalance**: Triggers when keys < MIN_KEYS. Borrow tries left then
  right sibling. Merge as fallback. `shrink_root` handles empty root.
- **Node encoding**: Encode and decode process fields in identical order for
  both Leaf and Internal. Round-trip tests exist.
- **Iterator**: Seek logic for Included/Excluded/Unbounded verified at both
  internal and leaf levels. `advance_leaf` stack traversal correct.

### Versioning

**Files**: `strata/src/versioned/` (mod.rs, map.rs, handle.rs, diff.rs,
merge.rs)

- **Ref-count lifecycle**: create_branch (+1), delete_branch (cascade -1),
  commit (net 0 on old HEAD), merge (tgt.head net 0, src.head +1), rollback_to
  (+1 target, -1 old_head with protect-before-cascade ordering).
- **DAG acyclicity (INV-V2)**: Commit IDs monotonically increasing; new commits
  reference only existing lower-numbered commits.
- **Source-wins merge (INV-V3)**: All 14 cases in the `three_way_merge` decision
  matrix verified. Test coverage exercises all conflict types.
- **Rollback (INV-V4)**: Only changes its own branch state. Uses
  increment-before-decrement to protect shared ancestors.
- **Dirty flag (INV-V5)**: Set in `decrement_ref` before cascade, cleared
  after. `gc()` checks flag and rebuilds if needed.
- **Commit immutability (INV-V6)**: Only `ref_count` modified on existing
  commits via increment_ref/decrement_ref. Root/parents/id/timestamp never
  mutated.

### Merkle Tries (MPT + SMT)

**Files**: `strata/src/trie/` (all subdirectories)

- **Proof soundness (INV-T1)**: Both MPT and SMT proof generation and
  verification use identical encoding, hashing (Keccak256), and path
  computation. Full encode/decode round-trip verified.
- **Nibble path (INV-T2)**: high = `byte >> 4`, low = `byte & 0x0F` in
  `nibbles.rs:from_raw`. Consistent across all operations.
- **SMT EMPTY_HASH (INV-T3)**: `[0u8; 32]` compile-time constant used in all
  files. `hash_internal(EMPTY, EMPTY) = EMPTY` special case correctly
  implemented.
- **Cache versioning (INV-T5)**: Keyed by (instance_id, commit_id). Load
  verifies root_hash. Keccak256 checksum (truncated to 8 bytes) protects
  against corruption.
- **Non-existence proofs (INV-T6)**: MPT handles empty child slot and divergent
  extension/leaf. SMT handles empty subtree, divergent leaf, and divergent
  internal node.

### Slot Index

**Files**: `strata/src/slotdex/` (all .rs files)

- **Tier boundaries**: `floor_align` is pure arithmetic (`self / base * base`),
  deterministic. Range transitions use `left_bound..right_bound` (exclusive
  right), then `left_bound = right_bound`. No gaps, no overlaps.
- **Pagination**: Slot-based skip counting, not numeric offset. Stable cursor.
- **DataCtner encoding**: Deterministic (BTreeSet sorted order,
  length-prefixed keys).
- **Small-to-Large upgrade**: Correct threshold check with dedup.
- **Finding**: insert path missing `ensure_cache()` after deserialization
  (see HIGH finding above).

### DAG Collections

**Files**: `strata/src/dagmap/` (all .rs files)

- **Acyclicity**: `get()` has MAX_DEPTH=1024 guard. Parent links only point
  backward by construction (child created after parent).
- **Node ID allocation (INV-DG1)**: Mutex held for full read-increment-write.

- **Edge operations**: `remove()` stores empty sentinel, `get()` treats empty as

- **Unsafe code**: 4 blocks total, all with valid SAFETY comments.
